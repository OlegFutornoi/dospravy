# План покриття: створення замовлення (business)

## Scope

- Розділ продукту: `business`
- Флоу: створення замовлення у майстрі `/order`
- Вимога виконання: один і той самий тест/спек має працювати на `dev`, `stage`, `prod` через `APP_AREA=business` і `TEST_ENV`.

## Передумови

- Авторизація у кабінет через телефон + OTP.
- Після входу користувач потрапляє на `/workspace`.
- Дані для авторизації беруться з `data/<env>/business/auth.json`.
- Дані для заповнення майстра беруться з `data/<env>/business/test-data.json` (секція `order.create`).

## Основний сценарій (E2E позитивний)

### 1. Перехід у майстер створення замовлення

1. Відкрити `/workspace`.
2. Перейти на `/order/` (або відкрити сторінку створення замовлення через навігацію, якщо така є).
3. Переконатись, що відкрився майстер `Створення замовлення`.

### 2. Крок 1 — інформація про компанію (автопідвантаження)

Мета: дочекатися, поки поля з даними компанії заповняться автоматично.

1. Очікувати, що поле `Назва компанії` має непорожнє значення.
2. Очікувати, що поле `Короткий опис компанії` має непорожнє значення.
3. Натиснути `Наступний крок`.

### 3. Крок 2 — інформація про замовлення

1. Заповнити `Назва позиції`.
2. Заповнити `Що потрібно робити`.
3. Обрати кілька `Переваги замовлення` (кнопки-теги).
4. Обрати `Стать`.
5. Заповнити діапазон `Бажаний вік працівника`.
6. Обрати категорії кандидатів.
7. Натиснути `Наступний крок`.

### 4. Крок 3 — вимоги та досвід

1. Обрати `Ключові вимоги`.
2. Заповнити `Що з собою мати на зміну`.
3. Обрати `Необхідні документи від кандидата`.
4. За потреби обрати `Інші документи` (select).
5. Обрати `Досвід`.
6. Натиснути `Наступний крок`.

### 5. Крок 4 — місце роботи

1. Заповнити `Місто`, `Вулиця`, `Номер будинку`.
2. Обрати `Берег (для Києва)` (select).
3. Обрати `Найближче метро` (select).
4. Заповнити `Район`, `Орієнтири`, `Як дістатись`.
5. Натиснути `Наступний крок`.

### 6. Крок 5 — графік і години роботи

Правило дати:

- дата зміни має бути наступною від дати запуску тесту (наприклад, сьогодні `17.05` → у тесті виставляємо `18.05`).

1. Заповнити `Оплата (грн/год)`.
2. Встановити `Період роботи` на наступну дату (один день).
3. Встановити `Початок зміни` = `09:00`, `Кінець зміни` = `18:00`.
4. Заповнити `К-сть працівників`.
5. Натиснути `Замовити`.

### 7. Підтвердження створення

1. Переконатись, що відображається текст формату: `Замовлення № <номер> створено!`
2. Зчитати номер замовлення (для логів/артефактів).

## Негативні перевірки (мінімум для регресії)

- Крок 1: кнопка `Наступний крок` недоступна, доки автополя не завантажені (якщо так задумано).
- Крок 2: обов'язкові поля не дають перейти далі без заповнення.
- Крок 5: неможливо `Замовити`, якщо дата в минулому або не задана.

## Стабільність

- Заборонено статичні очікування.
- Очікування тільки через стани:
  - автополя компанії: `toHaveValue` з непорожнім значенням;
  - перехід кроків: перевірка заголовка/індикатора кроку;
  - створення: перевірка тексту `Замовлення № ... створено!`.
